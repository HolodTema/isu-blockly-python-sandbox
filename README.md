# Визуальный конструктор кода для сборки и подготовки данных

Аналог Scratch или trinket.io для сбора и предобработки данных. Событийно-ориентированное
программирование при помощи drag-and-drop блоков кода, которые автоматически конвертируются
в python и методы библиотек pandas, requests, json. Возможность выполнения и отладки
сгенерированного python-кода на сервере. Импорт-экспорт проекта в файл. Возможность создания
аккаунта для сохранения созданных проектов на сервере.

# Используемые технологии

Backend: Java, Spring Boot

Frontend: Html, CSS, JS, React, Blockly

# Функциональные требования

## Аутентификация и управление пользователями
1. Регистрация. Пользователь может создать учётную запись, указав email и пароль.
Система проверяет корректность email и сложность пароля (например, не менее 8 символов).
После регистрации отправляется подтверждение на email (опционально).

2. Вход в систему
Пользователь может войти, введя email и пароль.
При успешном входе создаётся сессия (access_token, refresh_token) для последующих запросов.

3. Предусмотрен выход из системы (завершение сессии).

4. Восстановление пароля. Пользователь может запросить сброс пароля по email.

## Управление проектами
1. Просмотр списка проектов. Авторизованный пользователь видит на главном экране
все свои сохранённые проекты в виде карточек. Для каждого проекта отображаются:
название, дата последнего изменения, краткое описание.

2. Создание нового проекта. Система создаёт пустой проект с начальным набором блоков
("начало", "импорт библиотек" и вывод на экран "hello world").

3. Удаление и редактирование проекта
Возможность переименовать проект. Возможность удалить проект (с подтверждением).

4. Сохранение проекта. Автоматическое или ручное сохранение текущего состояния проекта
на сервере (структура блоков и сгенерированный код).

## Редактор проекта

1. Экран разделён на две области: слева — палитра блоков и рабочее поле библиотеки Blockly,
справа — окно с отображением сгенерированного Python-кода. Код генерируется по кнопке
"сгенерировать код".

2. После нажатия кнопки "сгенерировать код" пользователь нажимает кнопку "запуск".
Система отправляет Python-код на сервер для выполнения в изолированной среде
(в Docker-контейнере). Когда код запускается на сервере, пользователь видит
статус "в процессе". После завершения выполнения пользователь видит stdout, stderr,
полученные в результате работы файлы и тд.

## Виды блоков

1. Пользователь может перетаскивать блоки из палитры на рабочую область.
Блоки соединяются между собой по логике программирования. Используется библиотека Blockly
от Google.

- print, if-elif-else, for, while

- создание-получение переменной

- создание-получение массива, получение-элемента-массива-по-индексу

- создание-получение-словаря, получение-элемента-словаря-по-ключу.

- импорт библиотек (requests, json, pandas)

- выполнение HTTP-запросов (GET, POST, PUT, DELETE) с настройкой параметров
(path, query, body)

- обработка body (обычно body в json формате) и status-code из http-ответов

- преобразование данных в pandas DataFrame и базовые операции
